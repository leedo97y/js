이벤트 루프

자바스크립트는 한번에 하나의 코드만 실행할 수 있다. 하짐나, 사용되는 환경은 동시에 많은 작업이 처리된다.
자바스크립트에서 이벤트 루프는 비동기 방식으로 동시성을 지원하는 방법


< 이벤트 루프 용어 >
* 프로세스
: 프로그램을 메모리에 할당

* 스레드
: 프로세스 내의 실행 단위 (일종의 실행단위)
==> 소리, 화면, 입력 등

* 스택 (자료구조)
: Last In First Out = 마지막으로 넣은것이 첫번째로 빠진다.
==> 모바일 네비게이션 (뒤로가기), 수건쌓기

* 큐 (자료구조)
: First In First Out = 처음으로 넣은것이 첫번째로 빠진다.
==> 자판기 음료, 선입선출



1. WEB API 
: 웹을 만들때 사용할 수 있는 모든 인터페이스
==> SetTimeout 은 여기에 위임

2. Task Queue 
: WEB API에서 비동기 작업들이 실행된 후 호출되는 콜백 함수들이 대기
==> 콜스택이 비어있다면 테스크 큐에 있던 콜백 함수들이 실행됨
 
3. Event Loop

<JS Engine>
4. Call Stack 
: 함수 처리를 저장하는 영역
==> 함수가 수건 쌓이듯이 쌓임 / 여기가 비어있어야 넘어감

Heap 
: 구조화 되어있지 않은 단순한 영역 (객체가 할당)
